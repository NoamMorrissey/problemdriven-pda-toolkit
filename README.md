# Problem-Driven AI Toolkit

> **Think first. Build second.**

An open-source toolkit that implements the [Problem-Driven AI](https://problemdriven.ai) methodology. It connects human problem-solving with AI-assisted code generation, ensuring that everything built traces back to a validated problem.

**The problem it solves:** AI coding agents are powerful builders, but they build whatever you tell them — even if it's wrong. Most failed projects don't fail because of bad code. They fail because the wrong thing was built. PDA structures the thinking that must happen *before* any code exists.

**Who it's for:** Product teams, founders, and engineers who want their AI-generated code to solve a real, validated problem — not a guess.

---

## 🔄 The Flow

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  /pda-problem  ──→  Problem Statement    ──→  ✅ Human approves │
│                                                                 │
│  /pda-solution ──→  Solution Brief       ──→  ✅ Human approves │
│                                                                 │
│  /pda-context  ──→  Context Specification──→  ✅ Human approves │
│                                                                 │
│  /pda-ai-build ──→  Build + Verify       ──→  ✅ Human approves │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

Four phases. Four commands. Four human approval gates. The AI does not move forward until you say so.

## 🚀 Quick Start

```bash
git clone https://github.com/problemdriven/pda-toolkit.git my-project
cd my-project

# Add your research evidence to research/, then:
/pda-problem
```

## 📁 File Structure

```
your-project/
├── docs/
│   ├── pda-problem.md              ← Problem Statement (generated by /pda-problem)
│   ├── pda-solution-brief.md       ← Solution Brief (generated by /pda-solution)
│   └── pda-context-spec.md         ← Context Specification (generated by /pda-context)
│
├── templates/
│   ├── pda-problem.template.md     ← Empty PS template
│   ├── pda-solution-brief.template.md  ← Empty SB template
│   └── pda-context-spec.template.md    ← Empty Context Spec template
│
├── .claude/agents/                 ← Claude Code agents
│   ├── pda-problem.md
│   ├── pda-solution.md
│   ├── pda-context.md
│   └── pda-ai-build.md
│
├── .github/prompts/                ← GitHub Copilot prompts
├── .cursor/rules/                  ← Cursor rules
│
├── .specify/                       ← Spec Kit artifacts (generated by /pda-ai-build)
├── pda-verification/               ← Verification reports (generated)
│
├── CLAUDE.md                       ← Claude Code project instructions
├── AGENTS.md                       ← Copilot / generic agent instructions
└── README.md
```

## 🤖 Multi-Agent Support

| Platform | Location | Format |
|---|---|---|
| **Claude Code** | `.claude/agents/pda-*.md` | Agents with frontmatter |
| **GitHub Copilot** | `.github/prompts/pda-*.md` | Prompt files |
| **Cursor** | `.cursor/rules/pda-*.mdc` | Rule files with frontmatter |

All formats encode the same behavior. The Claude Code agents are the canonical source.
